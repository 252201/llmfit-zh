# llmfit 汉化版

**成百上千的模型与提供商，只需一条命令即可找出最适合你硬件运行的那一个。**

这是一个终端工具，可以根据你系统的 RAM、CPU 和 GPU，为你匹配并推荐最合适的 LLM 模型。它会自动检测你的硬件，从模型质量、运行速度、内存匹配度以及上下文长度等多个维度进行评分，并告诉你哪些模型在你的机器上能真正流畅运行。

本项目为 `llmfit` 的**完全中文汉化版**。主要包含了交互式的终端界面（TUI）和传统的命令行模式（CLI），并支持多 GPU 设置、MoE 架构、动态量化选择、速度预估以及本地运行提供商接入（例如 Ollama, llama.cpp, MLX）。

---

## 一键安装

你现在可以通过以下一条命令直接在终端里安装编译好这个汉化版（这需要你的电脑里已安装 [Rust](https://rustup.rs/)）：

```sh
cargo install --git https://github.com/252201/llmfit-zh.git llmfit
```

安装完成后，直接在终端里输入 `llmfit` 即可启动交互式的汉化版界面。

## 使用说明

### TUI 终端界面（默认）

直接在终端输入：
```sh
llmfit
```
即可启动交互式终端界面。界面上方会显示你的系统硬件配置（CPU、RAM、GPU、VRAM及后端），下方会列出所有模型并按综合评分排序。每一行展示了模型的评分、预估速度（tok/s）、最佳量化推荐、运行模式、内存使用率以及推荐用途。

**常用快捷键：**

| 按键 | 功能 |
|---|---|
| `↑` / `↓` 或 `j` / `k` | 上下导航模型 |
| `/` | 进入搜索模式（可通过名称、提供商、参数量、用途进行搜索） |
| `Esc` 或 `Enter` | 退出搜索模式 |
| `Ctrl-U` | 清空搜索词 |
| `f` | 切换匹配度过滤：全部、可运行、完美、良好、勉强等 |
| `a` | 切换可用性过滤：全部、GGUF可用、已安装 |
| `s` | 切换排序依据：评分、参数、内存%、上下文、日期、用途 |
| `v` | 进入多选模式（Visual mode）进行批量选中 |
| `V` | 进入单列过滤模式（Select mode） |
| `t` | 切换界面主题颜色（会自动保存） |
| `p` | 打开规划模式（估算运行该模型所需的硬件） |
| `P` / `U` / `C` | 分别打开：提供商、用途、能力 的过滤弹窗 |
| `m` | 标记选中的模型以进行对比 |
| `c` | 打开对比视图 |
| `d` | 下载选中的模型 |
| `r` | 刷新已安装的模型 |
| `Enter` | 展开选中模型的详细信息 |
| `q` | 退出程序 |

更多类似 Vim 的操作模式（如多选、列过滤等），请在界面下方状态栏查看实时快捷键提示。

### CLI 命令行模式

如果你更喜欢查看一次性输出的表格，可以使用 `--cli` 标志或各种子命令：

```sh
# 以表格形式列出所有匹配的模型
llmfit --cli

# 只显示完美匹配的顶级模型，限制前 5 个
llmfit fit --perfect -n 5

# 查看检测到的系统硬件信息
llmfit system

# 列出数据库中的所有模型
llmfit list

# 搜索特定模型（按名称、提供商或大小）
llmfit search "llama 8b"

# 查看某个模型的详细信息
llmfit info "Mistral-7B"

# 获取适合你的前 5 个推荐模型（JSON 输出，非常适合自动化脚本）
llmfit recommend --json --limit 5

# 针对“编程”用途推荐前 3 个模型
llmfit recommend --json --use-case coding --limit 3
```

## 硬件检测说明

- **macOS（Apple Silicon M系列）**：完美支持。通过 `system_profiler` 检测统一内存，VRAM 等于系统 RAM，使用 Metal 后端加速。
- **Windows / Linux**：支持通过 `nvidia-smi` 或 `rocm-smi` 等检测独立显卡及 VRAM。
- **强制覆盖 GPU 内存**：如果自动检测失败，可以使用 `--memory` 参数手动指定，例如：`llmfit --memory=24G` 或 `llmfit --memory=32000M`。
